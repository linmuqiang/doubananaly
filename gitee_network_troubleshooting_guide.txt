# Gitee网络连接故障排除指南

## 问题分析
根据错误信息：`fatal: unable to access 'https://gitee.com/fu-mengqiang/douban-analysis.git/': Failed to connect to gitee.com port 443 via 127.0.0.1 after 2077 ms: Could not connect to server`

这表明您的电脑无法通过端口443连接到gitee.com，可能是由以下原因导致：
1. 网络连接问题
2. 防火墙或安全软件限制
3. 代理设置问题
4. DNS解析问题

## 详细故障排除步骤

### 步骤1：测试基本网络连接
1. 打开命令提示符(cmd.exe)以管理员身份运行
2. 输入以下命令测试DNS解析：
   ```
   nslookup gitee.com
   ```
   如果显示IP地址（如212.64.63.190），说明DNS解析正常

3. 测试网络连通性：
   ```
   ping 212.64.63.190
   ```
   如果能收到回复，说明网络层连接正常

### 步骤2：检查防火墙和安全软件
1. 暂时禁用Windows防火墙：
   - 打开控制面板 → 系统和安全 → Windows Defender防火墙 → 关闭Windows Defender防火墙
   - 尝试再次推送代码
   - 如果成功，需要为Git添加防火墙例外规则

2. 检查第三方安全软件（如360安全卫士、腾讯电脑管家等）的网络拦截设置，确保Git被允许访问网络

### 步骤3：配置Git使用HTTP代理（如果您使用代理上网）
如果您的网络需要通过代理访问外部网站，请配置Git使用代理：
```
# 设置HTTP代理
git config --global http.proxy http://代理服务器IP:端口

# 设置HTTPS代理
git config --global https.proxy https://代理服务器IP:端口

# 如果不需要代理，清除代理设置
git config --global --unset http.proxy
git config --global --unset https.proxy
```

### 步骤4：切换到SSH连接方式
如果HTTPS连接始终失败，建议切换到SSH连接方式：

#### 4.1 生成SSH密钥对
1. 打开Git Bash或命令提示符
2. 输入以下命令生成SSH密钥（使用您的邮箱）：
   ```
   ssh-keygen -t ed25519 -C "984841296@qq.com"
   ```
3. 按Enter键接受默认保存位置
4. 设置密码（可选，建议设置）

#### 4.2 将公钥添加到Gitee
1. 查看公钥内容：
   ```
   type %USERPROFILE%\.ssh\id_ed25519.pub
   ```
2. 复制显示的全部内容
3. 登录Gitee → 点击头像 → 设置 → SSH公钥 → 粘贴公钥内容 → 保存

#### 4.3 修改Git远程仓库URL为SSH格式
```
# 先删除现有远程仓库
git remote remove gitee

# 添加SSH格式的远程仓库
git remote add gitee git@gitee.com:fu-mengqiang/douban-analysis.git

# 测试SSH连接
ssh -T git@gitee.com

# 推送代码
git push -u gitee master
```

### 步骤5：手动上传代码作为备选方案
如果上述方法都失败，您可以尝试手动上传代码：
1. 在Gitee网页上下载仓库的ZIP文件
2. 将您的项目文件复制到解压后的文件夹中
3. 在Gitee网页上直接上传文件

## 其他可能的解决方案
- 尝试使用手机热点连接网络，排除局域网限制
- 检查您的网络是否有IPv6支持问题，尝试禁用IPv6
- 更新Git到最新版本

## 验证连接成功的方法
成功连接后，您可以使用以下命令验证：
```
# 验证远程连接
git remote -v

# 尝试获取远程仓库信息
git fetch gitee
```

如果您在尝试上述步骤后仍然遇到问题，可能需要联系您的网络管理员或Gitee客服获取进一步帮助。

祝您成功上传项目！
